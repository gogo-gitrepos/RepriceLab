## What to automate (minimum)

### Smoke suite
Very fast checks that tell me the system is basically alive:

- App main page loads without errors.
- Login page renders correctly.
- Can log in with a valid test account and reach dashboard.
- API healthcheck or status endpoint returns success (200).

### E2E flows
Realistic end-to-end journeys, for example:

- **Login flow:** open main page ‚Üí go to login ‚Üí enter email/password ‚Üí reach dashboard ‚Üí assert dashboard key elements.

- **Forgot password** or any important account recovery flow (if available).

- `LoginPage`
- `DashboardPage`
- `PricingRulesPage`
- etc.

### Functional UI tests
More detailed tests on important components, such as:

- Form validations (email format, required fields, password rules).
- Error states (wrong password, invalid input).
- Navigation (header links: Features, Pricing, Contact, Login, etc.).
- Visibility of pricing cards, buttons and CTAs.

### Backend/API tests
Create a small but solid API test layer that:

- Hits auth/login endpoint and verifies correct responses.
- Tests main repricing-related endpoints (e.g. create/update rule, list rules, etc.).
- Verifies correct status codes, response schemas, and error handling.

Use a shared **API client helper** to avoid duplication.
